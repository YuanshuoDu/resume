## 一、FontAwesome 6 Free + XeLaTeX 最佳实践

### 核心发现

**问题根源**：之前 CI 失败是因为 fontawesome5.sty 指向系统路径，但 CI 环境没有该字体。

**解决方案**：把字体打包进项目

```
resume/
├── fonts/
│   ├── Font Awesome 6 Free-Solid-900.otf
│   ├── Font Awesome 6 Free-Regular-400.otf
│   └── Font Awesome 6 Brands-Regular-400.otf
└── fontawesome5.sty
```

**fontawesome5.sty 关键配置**：
```latex
\newfontfamily\FAFamilySolid[Path=fonts/]{Font Awesome 6 Free-Solid-900}
\newfontfamily\FAFamilyBrands[Path=fonts/]{Font Awesome 6 Brands-Regular-400}

\def\faGithub{{\FAFamilyBrands\symbol{"F092}}}
\def\faLinkedin{{\FAFamilyBrands\symbol{"F08C}}}
\def\faEnvelope{{\FAFamilySolid\symbol{"F0E0}}}
\def\faPhone{{\FAFamilySolid\symbol{"F095}}}
\def\faMapMarker{{\FAFamilySolid\symbol{"F3C5}}}
```

**fontspec Path= 选项是关键技术**：
- `\newfontfamily\FAFamilySolid[Path=fonts/]{Font Awesome 6 Free-Solid-900}`
- `Path=` 告诉 fontspec 去哪里找字体文件
- 不加 `Path=` 只用文件名，fontspec 会去系统字体目录找

### FA6 Brands vs Solid

| 字体 | 用途 | 图标示例 |
|------|------|---------|
| FA6 Free Solid | 实心图标 | Envelope, Phone, MapMarker |
| FA6 Brands | 品牌图标 | GitHub, LinkedIn, Twitter |

### 其他 FA6 Free 图标（简历可用）

```latex
\def\faGlobe{{\FAFamilySolid\symbol{"F0AC}}}        % 网站
\def\faGraduationCap{{\FAFamilySolid\symbol{"F19D"}}} % 教育
\def\faExternalLink{{\FAFamilySolid\symbol{"F08E"}}}  % 外部链接
```

## 二、LaTeX fontspec 深度研究

### fontspec 核心概念

```latex
\usepackage{fontspec}

% 设置英文字体（主字体）
\setsansfont{Source Sans Pro}[...options...]

% 设置中文字体
\setmainfont{Noto Serif CJK SC}

% 自定义字体族
\newfontfamily\HeaderFont{Myriad Pro}
```

### Fontspec + XeLaTeX vs LuaLaTeX

| 功能 | XeLaTeX | LuaLaTeX |
|------|---------|----------|
| 加载 OTF | ✅ | ✅ |
| 加载 TTF | ✅ | ✅ |
| OpenType features | ✅ 部分 | ✅ 全部 |
| 字体路径 `Path=` | ✅ | ✅ |
| 微调排版 | 仅 protrusion | 全部 |

### fontspec Path= 正确用法

```latex
% 相对路径（相对于 LaTeX 文档）
\newfontfamily\LatoFont[Path=fonts/]{Lato-Regular}

% 绝对路径
\newfontfamily\LatoFont[Path=/usr/share/fonts/truetype/lato/]{Lato-Regular}
```

**常见错误**：
```latex
% ❌ 错误：XeLaTeX 下 fontspec 无法解析相对路径
\newfontfamily\FAFamily{fonts/Font Awesome 6 Free-Solid-900}

% ✅ 正确：加 Path= 选项
\newfontfamily\FAFamily[Path=fonts/]{Font Awesome 6 Free-Solid-900}
```

## 三、简历 Typography 专业指南

### 2025-2026 简历字体趋势

**最佳简历字体（英文）**：

| 字体 | 类型 | 推荐度 | 适用场景 |
|------|------|--------|---------|
| **Lato** | Sans-serif | ⭐⭐⭐⭐⭐ | 技术岗位，现代感 |
| **Source Sans Pro** | Sans-serif | ⭐⭐⭐⭐⭐ | 专业、清晰 |
| **Fira Sans** | Sans-serif | ⭐⭐⭐⭐ | 技术/设计 |
| **Libre Baskerville** | Serif | ⭐⭐⭐⭐ | 创意/学术 |
| **Garamond** | Serif | ⭐⭐⭐⭐ | 传统行业 |

**简历字体组合建议**：
- 标题：Lato Bold 或 Source Sans Pro Bold
- 正文：Lato Regular 或 Source Sans Pro Regular
- 代码/技术：Fira Mono

### LaTeX 简历专业排版规则

**1. 字体层级**：
```latex
% 姓名 - 最大
{\Large\sffamily\bfseries 姓名}

% Section 标题
\section{Skills}  % 自动蓝色竖条

% 子标题
\textbf{岗位名称}  % 公司名 \hfill 日期
```

**2. 间距系统**（simpleresume.cls）：
```latex
\section间距: 1.1ex (上) / 0.35ex (下)
\item间距: itemsep=0.06cm, parsep=0.03cm
段落间距: 0.05cm
```

**3. 图标使用原则**：
- 同一行图标保持一致的视觉权重
- 图标颜色 = accent 颜色（保持品牌一致性）
- 图标大小匹配文字高度

## 四、fancyhdr 常见 Bug 与修复

### 问题：Header 残留文字

**症状**：右上角出现 "Certifications & Publication" 或其他 Section 标题

**原因**：fancyhdr 默认会把最后一个 `\section` 名称放到 header

**修复**：
```latex
\fancypagestyle{plain}{
  \fancyhead{}% ← 清空所有 header
  \fancyfoot[L]{...}
  \fancyfoot[C]{...}
  ...
}
```

**FA6 Free vs Pro**：
- Free: 2,000+ 图标
- Pro: 7,000+ 图标 + 变体 + 托管
- 简历使用 Free 足够

### CI 环境字体安装

**GitHub Actions Ubuntu Runner** 字体安装方式：
```yaml
- name: Install fonts
  run: |
    sudo apt-get update
    sudo apt-get install -y fonts-freefont-otf
    # 自定义字体放项目里
```

**最佳实践**：把字体打包进项目，不依赖 CI 环境
- 字体文件放在 `fonts/` 目录
- 用 `Path=` 引用相对路径
- 每次 git push 带上字体文件

## 七、关键结论

1. **FontAwesome + XeLaTeX** 完全可以正常工作，关键是用 `Path=` 引用项目内字体
2. **fancyhdr** 使用前一定要 `\fancyhead{}` 清空，否则会有残留
3. **简历字体** Lato > FreeSans > 默认字体
4. **图标颜色** 要统一用 accent 颜色，不能有的 accent 有的 muted
5. **Summary** 单段落不需要 leading bar（蓝色竖条），反而显得多余
